# 数据绑定库

例如，以下代码会调用 `findViewById()` 来查找 `TextView` widget 并将其绑定到`userName` `viewModel` 变量：

```kotlin
findViewById<TextView>(R.id.sample_text).apply {
    text = viewModel.userName
}
```

或者

```xml
<TextView
    android:text="@{viewmodel.userName}" />
```

通过布局文件中的绑定组件，可以从中移除许多界面框架调用。这可以提高应用性能，并帮助防止内存泄漏和 null 指针异常。



## 使用可观察的数据对象

可观察类有三种类型： [字段](https://developer.android.com/topic/libraries/data-binding/observability?hl=zh-cn#observable_fields)、[集合](https://developer.android.com/topic/libraries/data-binding/observability?hl=zh-cn#observable_collections)和 [对象](https://developer.android.com/topic/libraries/data-binding/observability?hl=zh-cn#observable_objects) 。

```kotlin
class User {
    val firstName = ObservableField<String>()
    val lastName = ObservableField<String>()
    val age = ObservableInt()
}
```

```kotlin
ObservableArrayMap<String, Any>().apply {
    put("firstName", "Google")
    put("lastName", "Inc.")
    put("age", 17)
}
```

实现数据类 `BaseObservable` 负责在属性更改时发出通知。待办事项 为其分配 [`Bindable`](https://developer.android.com/reference/android/databinding/Bindable?hl=zh-cn) 注解 传递给 getter，并调用 [`notifyPropertyChanged()`](https://developer.android.com/reference/android/databinding/BaseObservable?hl=zh-cn#notifypropertychanged) 方法：

```kotlin
class User : BaseObservable() {

    @get:Bindable
    var firstName: String = ""
        set(value) {
            field = value
            notifyPropertyChanged(BR.firstName)
        }

    @get:Bindable
    var lastName: String = ""
        set(value) {
            field = value
            notifyPropertyChanged(BR.lastName)
        }
}
```

数据绑定在模块软件包中生成一个名为 `BR` 的类，其中包含 用于数据绑定的资源的 ID。`Bindable` 注解 在编译期间在 `BR` 类文件中生成一个条目。

## 生命周期感知型对象

### 使用 StateFlow

```kotlin
class ViewModelActivity : AppCompatActivity() {
    override fun onCreate(savedInstanceState: Bundle?) {
        // Inflate view and obtain an instance of the binding class.
        val binding: UserBinding = DataBindingUtil.setContentView(this, R.layout.user)

        // Specify the current activity as the lifecycle owner.
        binding.lifecycleOwner = this
    }
}
```

```kotlin
class ScheduleViewModel : ViewModel() {

    private val _username = MutableStateFlow<String>("")
    val username: StateFlow<String> = _username

    init {
        viewModelScope.launch {
            _username.value = Repository.loadUserName()
        }
    }
}
```

在您的布局中，将 `ViewModel` 对象的属性和方法分配给 相应的视图，如以下代码所示：

```kotlin
<TextView
    android:id="@+id/name"
    android:text="@{viewmodel.username}" />
```

每当用户的名称值发生变化时，界面都会自动更新。

## 绑定适配器

### 提供自定义逻辑

一些属性需要自定义绑定逻辑。例如，没有 `android:paddingLeft` 属性的 setter 方法。而是提供了 `setPadding(left, top, right, bottom)` 方法。具有如下参数的静态绑定适配器方法： [`BindingAdapter`](https://developer.android.com/reference/androidx/databinding/BindingAdapter?hl=zh-cn) 注解可让您自定义属性 setter 的调用方式。

Android 框架类的属性已有 `BindingAdapter` 注释。以下示例显示了 `paddingLeft` 属性：

```kotlin
@BindingAdapter("android:paddingLeft")
fun setPaddingLeft(view: View, padding: Int) {
    view.setPadding(padding,
                view.getPaddingTop(),
                view.getPaddingRight(),
                view.getPaddingBottom())
}
```

您还可以使用可接收多个属性的适配器，如示例：

```kotlin
@BindingAdapter("imageUrl", "error")
fun loadImage(view: ImageView, url: String, error: Drawable) {
    Picasso.get().load(url).error(error).into(view)
}
```
