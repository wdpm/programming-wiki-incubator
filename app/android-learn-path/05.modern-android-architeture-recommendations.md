# 架构建议

## 界面层

以下代码段简要说明了如何以生命周期感知型方式收集界面状态：

View：

```kotlin
class MyFragment : Fragment() {

    private val viewModel: MyViewModel by viewModel()

    override fun onViewCreated(view: View, savedInstanceState: Bundle?) {
        super.onViewCreated(view, savedInstanceState)

        viewLifecycleOwner.lifecycleScope.launch {
            viewLifecycleOwner.repeatOnLifecycle(Lifecycle.State.STARTED) {
                viewModel.uiState.collect {
                    // Process item
                }
            }
        }
    }
}
```

Compose：

```kotlin
@Composable
fun MyScreen(
    viewModel: MyViewModel = viewModel()
) {
    val uiState by viewModel.uiState.collectAsStateWithLifecycle()
}
```

## 测试

尽量采用虚假实现，而非模拟实现。参阅 [Android 文档中的“使用测试替身”](https://developer.android.com/training/testing/fundamentals/test-doubles?hl=zh-cn)。

测试 StateFlow。测试 `StateFlow` 时： [对 `value` 属性进行断言](https://developer.android.com/kotlin/flow/test?hl=zh-cn#stateflows) 。如果使用 `WhileSubscribed`，您应该[创建一个 `collectJob`](https://developer.android.com/kotlin/flow/test?hl=zh-cn#statein) 。
